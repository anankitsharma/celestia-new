## Critical rules baked into every field

1. **Banned-word zero count** in app name + subtitle + first 200 chars of description + keywords:
   `horoscope`, `daily horoscope`, `zodiac`, `fortune`, `tarot`, `manifest`, `manifesting`, `destiny`, `predict`, `prediction`, `Mercury retrograde`, `cosmic`, `divine`, `sacred`, `oracle`, `crystal`, `numerology`.

2. **Approval-first: zero astrology vocabulary in metadata.** No `astrology`, `astrologer`, `birth chart`, `planetary`, or any related stem appears anywhere in the description, promo, subtitle, name, or keywords. The in-app astrology toggle (hidden by default, labeled `Show astrology details`) is a real feature but is not advertised in store metadata — there is no requirement to disclose every opt-in toggle in the listing.

3. **No keyword duplication** between Title, Subtitle, and Keyword Field. Apple's Full Text Search auto-combines across fields, so the same word in two places wastes the budget. Captions are the exception — they *should* reinforce title/subtitle keywords.

4. **Screenshot captions are now Tier 1 ranking signals** (post June 2025). Every screenshot has a single bold caption at the top of the frame, OCR-readable, in 3–7 words.

5. **Categories are locked** — Primary: `Health & Fitness > Mindfulness`. Secondary: `Lifestyle`. Never Entertainment. Never Reference. Never Social Networking.

## How to use this directory

**Order of operations the day you submit:**

1. Open `validation.md` — verify every field passes char limits. If anything's red, fix in `metadata.md` before going further.
2. Open `metadata.md` — paste each field into App Store Connect verbatim from the fenced blocks.
3. Open `screenshot-captions.md` — verify the 7 captions in the live generator (`plan/screenshots/generator/`) match the spec, then run "Export all (7)" at the 6.9" preset to produce the 1320×2868 PNGs.
4. Open `whats-new.md` — paste into App Store Connect → Version Information → What's New.
5. The Reviewer Notes message stays at `iOS-version/plan/Narrative-ASO/Reviewer-Message.md` — paste that verbatim into App Review Information → Notes.

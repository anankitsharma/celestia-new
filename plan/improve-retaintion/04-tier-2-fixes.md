# Tier 2 — Deepen the fix

Ship after Tier 1 lands and you have ~14 days of paid-cohort data to verify Tier 1 worked. These are valuable but not as load-bearing as A/B/C.

## Fix D — "Pro insight" daily-briefing section

### Why
Pro users see the same daily briefing as free users. There's no daily moment where the subscription delivers extra. The user pays $49.99/year and sees the same morning content every day.

### What
Add a single extra section to `HomeScreen` daily content that only renders for `isPro === true`. Generated by a new Gemini prompt that produces ~3 sentences of *deeper* content tied to that day's transits — psychological, behavioral, specific.

Position it: between the navigator briefing and the life-area cards. Visually distinct (gold-accent border, "PRO INSIGHT" label).

### Implementation outline
- New `geminiService.generateProDailyInsight(chart, transitContext, journalContext)` returning `{ headline, body }`
- HomeScreen: when `isPro && tab === 'today'`, fetch + cache for the day
- Cache key: `${profileId}_pro_insight_${dateLabel}`
- Falls back gracefully (just hides) if Gemini call fails

### Effort
~3 hours. Mirrors the briefing-mode rotation (P2.1) pattern.

### Acceptance
- [ ] Pro users see a "PRO INSIGHT" card between navigator and life areas
- [ ] Free users see no change (control)
- [ ] Content cached per (user, day); doesn't re-generate on re-mount
- [ ] Distinct visual treatment so it feels earned

## Fix E — Trial-cancel-flow variants

### Why
The cancel flow we shipped (P3.1) has 8 reason chips. They were designed for established subscribers. Users cancelling on trial day 5-6 have a different mental model:
- "Forgot I had a trial"
- "Tried it briefly, not worth $X"
- "Auto-renewal anxiety"
- "Didn't have time to try it properly"

None of these map cleanly to existing `too_expensive` / `not_using` / `missing_feature` etc.

### What
Detect at CancelFlowScreen mount whether the user is in trial vs. paid:

```js
const isTrialCancel = customerInfo?.entitlements?.active?.['Celestia Pro']?.periodType === 'TRIAL';
```

If `isTrialCancel`, show a different reason set:

```
○ Didn't have time to try it properly
○ Tried it, not what I was hoping for
○ Forgot I had a trial — don't want to be charged
○ The trial was helpful but I'm not ready to commit
○ Privacy concerns
○ Switching to another app
○ Other
```

Each gets a save offer matched to trial mental model. For example:
- "Didn't have time" → "Want us to extend your trial by 7 days?" (requires App Store Connect promo offer setup; otherwise: "Your trial ends in N days — here's the highest-leverage feature to try right now")
- "Forgot I had a trial" → "We hear that a lot. We just sent you a calendar reminder so you can decide on your own time." (purely informational; respects autonomy)

### Effort
~2 hours. Reuse existing CancelFlowScreen infrastructure; add a parallel reason set + offer matrix.

## Fix F — Pro week-1 recap card

### Why
We shipped a "first week recap" for any user (GAP-5) at D7 streak. There's no equivalent that anchors specifically to **Pro features used in the first 7 days of being Pro**.

### What
At day 7 of being Pro, generate a recap card that lists:
- Reports generated (with names)
- Deep-dives done (with planet names)
- Partners added to Circle
- Chats sent
- Any badges unlocked specifically from Pro features

Frame it as: "Your first week as a Pro. Here's what you've explored."

If the user has used 0-1 Pro features by Day 7, the card flips: "Most Pros use [feature] in their first week. Want to try?" — linking to that feature.

### Effort
~3 hours. Mirrors GAP-5 first-week recap pattern but with Pro-specific data.

## Tier 2 sequencing

Ship in this order: **D → F → E**.

- D delivers daily perceived value (highest ongoing impact)
- F catches the user at a meaningful moment (Day 7)
- E catches users who're already committing to leave; smaller funnel but high-leverage per user

## Tier 2 success metrics

Tier 2 is verified by:
1. **D30 paid retention rate** climbs (the headline metric)
2. **Pro-feature usage diversity** — # of paid users who've tried 3+ Pro features in their first 30 days (rises 30%+)
3. **Trial→paid conversion rate** rises (better save offers in E + better discovery in D + F)
